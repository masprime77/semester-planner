# Updating the app icon

The macOS app icon **and** the DMG volume icon both come from a single file,
**`packages/desktop/assets/icon.icns`**, compiled from
**`packages/desktop/assets/icon.png`**. Nothing else needs changing —
`packages/desktop/package.json` already points both `build.mac.icon` and
`build.dmg.icon` at `assets/icon.icns`.

All paths below are relative to the desktop workspace
(`packages/desktop/`); the icon scripts live there, so run them with
`--workspace @lectio/desktop` from the repo root (the root `package.json` has no
`icon` alias).

All tooling is built into macOS (`sips`, `iconutil`) — no external apps required.

## TL;DR

```bash
# 1. Replace packages/desktop/assets/icon.png with your new 1024×1024 artwork.
# 2. Rebuild the .icns (from the repo root):
npm run icon --workspace @lectio/desktop
# 3. Commit both files:
git add packages/desktop/assets/icon.png packages/desktop/assets/icon.icns
git commit -m "chore: update app icon"
```

The new icon then ships with the next release (see [Shipping](#shipping-the-new-icon)).

## The source PNG

`packages/desktop/assets/icon.png` must be:

- **1024×1024** pixels (PNG; transparency is fine).
- Designed as a **full-bleed rounded square** — macOS does **not** round app
  icons for you, so the artwork should already include its own corners/shape.

Keep editable masters (Figma/Sketch/Affinity exports, layered PNGs, etc.) in
`packages/desktop/assets/logo_raw/` — that folder is gitignored.

## What `npm run icon` does

`npm run icon --workspace @lectio/desktop` runs
[`build-icns.sh`](../packages/desktop/assets/build-icns.sh), which:

1. Creates a temporary `icon.iconset/` folder.
2. Uses `sips` to downscale `icon.png` into every size macOS needs, with the
   exact filenames `iconutil` expects:

   | File in the iconset            | Pixels |
   | ------------------------------ | ------ |
   | `icon_16x16.png`               | 16     |
   | `icon_16x16@2x.png`            | 32     |
   | `icon_32x32.png`               | 32     |
   | `icon_32x32@2x.png`            | 64     |
   | `icon_128x128.png`             | 128    |
   | `icon_128x128@2x.png`          | 256    |
   | `icon_256x256.png`             | 256    |
   | `icon_256x256@2x.png`          | 512    |
   | `icon_512x512.png`             | 512    |
   | `icon_512x512@2x.png`          | 1024   |

3. Runs `iconutil -c icns icon.iconset -o icon.icns`.
4. Deletes the temporary iconset.

### Doing it by hand (equivalent)

```bash
cd packages/desktop/assets
rm -rf icon.iconset && mkdir icon.iconset
sips -z 16 16     icon.png --out icon.iconset/icon_16x16.png
sips -z 32 32     icon.png --out icon.iconset/icon_16x16@2x.png
sips -z 32 32     icon.png --out icon.iconset/icon_32x32.png
sips -z 64 64     icon.png --out icon.iconset/icon_32x32@2x.png
sips -z 128 128   icon.png --out icon.iconset/icon_128x128.png
sips -z 256 256   icon.png --out icon.iconset/icon_128x128@2x.png
sips -z 256 256   icon.png --out icon.iconset/icon_256x256.png
sips -z 512 512   icon.png --out icon.iconset/icon_256x256@2x.png
sips -z 512 512   icon.png --out icon.iconset/icon_512x512.png
cp icon.png icon.iconset/icon_512x512@2x.png   # 1024
iconutil -c icns icon.iconset -o icon.icns
rm -rf icon.iconset
```

## Generating a placeholder icon (optional)

To regenerate the simple flat "SP" placeholder `icon.png` (e.g. to tweak its
color/letters), edit
[`generate-icon.js`](../packages/desktop/assets/generate-icon.js) and run it
with Electron's canvas (workspace deps are hoisted to the repo-root
`node_modules`, so run from the repo root):

```bash
node_modules/.bin/electron packages/desktop/assets/generate-icon.js   # rewrites icon.png
npm run icon --workspace @lectio/desktop                              # then rebuild the .icns
```

## DMG background (separate from the icon)

The DMG window's background is a different asset,
`packages/desktop/assets/dmg-background.png` (+ `@2x`), generated by
[`generate-dmg-background.js`](../packages/desktop/assets/generate-dmg-background.js):

```bash
node_modules/.bin/electron packages/desktop/assets/generate-dmg-background.js
```

## Files involved

| File | Role |
| ---- | ---- |
| `packages/desktop/assets/icon.png` | 1024×1024 source artwork (committed) |
| `packages/desktop/assets/icon.icns` | Built icon used by the app + DMG (committed) |
| `packages/desktop/assets/build-icns.sh` | `icon.png → icon.icns` (`npm run icon`) |
| `packages/desktop/assets/generate-icon.js` | Optional placeholder `icon.png` generator |
| `packages/desktop/assets/dmg-background.png` / `@2x` | DMG window background |
| `packages/desktop/assets/generate-dmg-background.js` | DMG background generator |
| `packages/desktop/package.json` → `build.mac.icon`, `build.dmg.icon` | Point at `assets/icon.icns` |

## Shipping the new icon

- **See it locally:** `npm run build:mac`, open the `.app` in
  `packages/desktop/dist/`. If Finder still shows the old icon, the icon cache is
  stale — `killall Dock` (or log out / reboot).
- **For users:** the icon is baked into the build, so it reaches them with the
  next published release (bump the version, tag, release — see the README's
  *Releasing* section). Installed copies update when they get the new version.
